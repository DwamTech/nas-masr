# دليل استخدام صورة الوظائف الموحدة

تم تطبيق التعديلات لجعل صورة قسم الوظائف اختيارية، مع عرض صورة افتراضية موحدة يتم التحكم فيها من لوحة التحكم (أو عبر الـ API).

## 1. رفع الصورة الافتراضية (عن طريق Postman)

لتعيين الصورة التي ستظهر لجميع الوظائف، استخدم الـ Endpoint الجديد:

**الرابط:**
`POST {{base_url}}/api/admin/system-settings/upload-image`

**الـ Headers:**
- `Authorization`: `Bearer {{admin_token}}`
- `Accept`: `application/json`

**الـ Body (form-data):**
| Key | Type | Value | Description |
| --- | --- | --- | --- |
| `key` | Text | `jobs_default_image` | **ضروري جداً** |
| `image` | File | (اختار الصورة) | ملف الصورة (jpg, png, webp) |

---

## 2. التأكد من النتيجة

1.  بعد الرفع بنجاح، ستعود الـ API برابط الصورة الجديدة.
2.  افتح أي إعلان وظائف (قائمة أو تفاصيل) عبر:
    `GET {{base_url}}/api/v1/jobs/listings`
3.  ستجد أن حقل `main_image_url` يعرض الصورة التي قمت برفعها للتو، حتى للإعلانات التي لم يتم رفع صورة لها.

## 3. ملاحظات للمطور (Frontend)

- عند إنشاء إعلان وظيفة جديد (`POST /api/v1/jobs/listings`)، لا داعي لإرسال حقل `main_image` في الـ body.
- إذا لم يتم إرسال صورة، سيتم قبول الإعلان وعرض الصورة الافتراضية تلقائياً.
- إذا قام الأدمن بتغيير الصورة الافتراضية في المستقبل، ستتغير فوراً في جميع إعلانات الوظائف في التطبيق دون الحاجة لتحديث التطبيق.
